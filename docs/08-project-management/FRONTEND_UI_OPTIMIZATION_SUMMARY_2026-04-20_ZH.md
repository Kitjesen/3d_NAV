# 前端 UI 优化总结（2026-04-20）

本文件记录 `ui-polish-theme-2026-04-20` 分支中的前端界面优化内容。该分支主要聚焦 MapPilot Web 控制台的视觉一致性、日间/暗黑主题、浮窗布局稳定性，以及若干交互细节修复。

## 优化范围

- 前端目录：`web`
- 主要页面：控制台、场景视图、SLAM 页面
- 主要目标：提升界面美观度、减少遮挡和重叠、增强日间模式可读性、修复前端交互问题

## 主要优化内容

### 1. 控制台右侧面板组合化

- 将原本独立的 `LocalizationCard` 和 `MiniMap` 合并为一个右侧控制面板组。
- 统一 SLAM / GNSS / 融合面板与 Mini Map 的边框、圆角、头部高度、玻璃材质和阴影。
- 解决右侧 Mini Map 因独立浮窗坐标或浏览器尺寸变化导致显示不完整的问题。

### 2. 浮窗布局响应式修复

- 为 `FloatingWidget` 增加基于父容器的边界约束，避免浮窗跑出可视区域。
- 为控制台主浮窗增加响应式布局计算，统一管理相机、右侧控制面板和聊天区的位置与尺寸。
- 升级浮窗布局本地存储版本，自动丢弃已失效的旧布局坐标。
- 支持为浮窗单独配置拖拽提示条位置，右侧控制面板的拖拽条移动到两块卡片交界处，避免遮挡 tab 文案。

### 3. 暗黑 / 日间模式切换

- 接入 `useTheme` 和 `ThemeToggle`，在 Topbar 中新增日间/暗黑切换按钮。
- 主题选择持久化到 `localStorage`，刷新后保留用户选择。
- 新增日间模式全局设计 token，包括背景、文字、边框、玻璃面板、阴影和状态色。
- 让 Three.js 场景背景跟随主题切换。

### 4. 日间模式细节修复

- 修复场景页部分深色浮层在日间模式下过暗的问题，包括：
  - 顶部图层按钮组
  - 左上角机器人状态浮层
  - 顶部场景操作提示
  - 右下角相机 PiP
  - 重定位展开面板
- 修复日间模式下“保存地图”按钮文字不可见的问题。
- 调整底部状态栏日间模式颜色，使文字和数值更清晰。

### 5. Mini Map 空状态修复

- 修复无定位数据时 Mini Map 仍绘制机器人箭头，导致“无定位数据”文字与图标重叠的问题。
- 无 odometry 时 Mini Map 只绘制淡网格，不再绘制机器人箭头。

### 6. 场景页交互修复

- 修复“清除点云”按钮的行为：现在只清除浏览器当前显示的 live point cloud，不再请求后端 `/api/v1/map_cloud/reset`。
- 避免后端未启动时点击“清除点云”产生额外 proxy error。
- 保留后续 WebSocket 新点云帧自动重新显示的行为。

### 7. 字体和状态显示优化

- 优化底部状态栏字体，从偏终端感的大号 mono 字体调整为更柔和的 UI 字体，并保留数字等宽对齐。
- 调小 SLAM 页面中 Localizer 状态词字号，避免“停止”等中文状态词过大。
- 调整若干状态标签、空状态和面板文案的视觉层级。

### 8. 前端说明与环境文档

- 新增前端中文使用说明。
- 新增前端 conda 环境配置。
- 新增项目学习和 Mac 开发指南。
- 记录本轮 Codex 协作过程和前端基线修复历史。

- 开发环境未启动后端时，Vite 仍可能打印 `/api/v1/events`、`/api/v1/bag/status`、`/ws/cloud` 等 proxy error，这是后端服务未运行导致的开发期噪声，不影响纯前端 UI 预览。
- 当前分支只用于前端优化验证，不直接提交到主干。
