## 本次发现的 Bug / 坑(按严重度排序)

### 🔴 Critical
1. **SLAM 位置漂移炸裂** 观察到里程计 `(-6965.7, 23939.4)` 坐标发散
   - 原因:长时间运行(>1 小时)+ 机器人静置 + Fast-LIO2 在没有新 LiDAR 运动刺激时 IEKF 会让协方差无限变大
   - 应急:`sudo systemctl restart slam`(已做,恢复到 8.6Hz)
   - 追查:`BACKLOG P1-06 SLAM 漂移守卫` 已存在相关 commit `b4919a8/643ee96`,但对这种"静置漂移"没触发 — **需要加一条"静置也监测 odom magnitude 突变"的守卫**

### 🟡 Major
2. **防火墙 5050/8090 默认 DROP** `ROBOT_REMOTE` iptables chain 只放 22/2020/50051/50052,Dashboard 外部访问被挡
   - 解决:已手动 `iptables -I` 开 5050 + 8090(但 **重启会丢**,需写入 /etc/iptables/rules.v4 持久化)
   - **action item**:给部署脚本加 iptables 持久化规则

3. **bag API NameError** 第一版 bag route 忘记 `import os`,`/api/v1/bag/status` 一调就 500 崩 gateway
   - 已修(`6789633`),每个 handler 顶部 `import os as _os`
   - 教训:pre-commit hook 没跑真 HTTP 冒烟,syntax 过了不等于 runtime 过了

4. **disk_free 查错分区** S100P `/` 是 50GB eMMC 剩 5GB,`~/data` 是 240GB NVMe 剩 222GB,bag 实际写在 `~/data/bags/` — 但 status API 报的是 `/`
   - 已修(`11f2b79`)

### 🟢 Minor
